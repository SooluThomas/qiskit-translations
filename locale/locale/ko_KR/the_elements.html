키스킷 구성 요소¶

테라¶

테라, 〈대지〉 엘리먼트는 나머지 키스킷 하위 구조들이 놓어지는 기반을 형성한다. 테라는 펄스와 회로 단위에서 양자 프로그램을 구성하는 기반을 제공하여서 특정 장치의 제한 아래서 최적화를 할 수 있게 해주고 다른 원격 접속된 장치 상에서 여러 실험들이 일괄 처리되는 것을 관리할 수 있게 해 준다. 테라는 원하는 최종 사용자 경험 및 최적화 계층, 펄스 스케줄링과 후위 처리장치 간의 커뮤니케이션 등을 효율적으로 다룰 수 있게 해주는 인터페이스를 정의한다.

키스킷 테라는 여섯가지 중요한 모듈로 구성되어 있다.

  1. 회로 양자 회로란 큐빗으로 이루어진 레지스터에 연속적인 양자 연산(보통 게이트들)을 통해 이루어지는 양자 컴퓨팅을 위한 모델이다. 보통 양자 회로는 \(|0,…,0>\) 상태에 있는 큐비트들로 시작되어 전통적인 컴퓨터로는 표현되기 힘든 상태의 큐비트들로 발전해 나간다. 양자 상태에 대한 정보를 알아내기 위해서는 결과값(양자 시스템의 근본적인 성질 때문에 임의값을 가질 수 있다)들을 효과적으로 표현할 수 있는 고전적인 레지스터들로 옮겨주는 측정이 필요하다.

  2. 펄스 펄스 스케줄은 (실험적인 입력 라인인) 채널에 적용되는 양자 실험으로 보내지는 펄스들의 집합이다. 이는 회로 보다 하위 레벨이며 회로에 있는 각각의 게이트는 펄스들의 집합으로 표현되어진다. 이 단계에서는 에러를 줄이기 위해 다양한 동적 분리(dynamic decoupling), 에러 완화(error mitigation), 최적 펄스 형상(optimal pulse shape)과 같은 방식으로 실험들을 디자인한다.

  3. 트랜스파일러 양자 컴퓨터 연구의 중심 파트는 어떻게 양자 회로를 실제 장치에서 구동시킬지를 알아내는 것이다. 이러한 장치들에서는 실험적인 에러와 위상 결집 (coherence)가 약해져 계산 과정동안 연산 에러를 만들게 된다. 따라서 탄탄한 실행을 위해서는 양자 회로의 게이트의 숫자를 줄이고 전체적인 연산 시간을 줄이는 것이 필수적이다. 트랜스파일러는 사용자들이 그들의 알고리즘을 위해 최적화를 시험해보고 더 나은 양자 회로를 발견하기 위한 경로 매니저의 개념을 도입하였다. 우리는 그것을 트랜스파일러라 부르는데 왜냐하면 그 연산의 결과 역시 또 하나의 회로이기 때문이다.

  4. 공급자들 사용자들이 후위 처리 장치에서 돌아갈 회로를 만들고 나면, 그들은 만들어진 회로를 다룰 손쉬운 방법이 필요하다. 테라에서는 다음의 네 가지 파트를 가지고 이것을 이루어낸다.

    1. 공급자 다른 후위 처리 장치의 그룹들에 대한 접근을 제공하는 개체다 (예를 들어, IBM 양자 경험 시스템(IBM Quantum Experience) 을 통해 제공되는 후위 처리 장치들). 공급자는 그러한 후위 처리 장치와 상호 작용하고, 예를 들면 사용 가능한 것을 찾거나, 특정 후위 처리 장치의 인스턴스를 찾도록 한다.

    2. 후위 처리 장치 는 실제 양자 컴퓨터나 시뮬레이터를 나타내는데, 양자 회로를 실행하고 결과를 출력하는 것을 담당한다. 이는 양자 객체(qobj)`를 입력으로 받아들여 `기본작업(BaseJob) 객체를 출력하는 실행 메소드를 가지고 있다. 이 객체는 작업이 끝났을 때 후위 처리 장치로부터 결과를 불러들이는 일을 비동기적으로 하도록 해준다.

    3. 작업 인스턴스는 제출된 작업을 위한 “티켓” 같이 생각될 수 있다. 이는 특정 시간에 (예를 들어, 작업이 대기열에 있거나, 실행, 또는 실패한 경우) 연산 상태를 알아내고, 작업에 대한 통제를 가능하게 해준다.

    4. 결과. 작업이 끝나면, 테라는 원격 후위 처리 장치로부터 result = job.result()`라는 명령을 통해 결과를 얻을 수 있게 한다. 이런 result 객체는 양자 정보를 가지고 있으며 그것과 상호 작용할 수 있는 가장 일반적인 방법은 `result.get_counts(circuit) 명령어를 사용하는 것이다. 이 메소드는 사용자들이 양자 회로로부터 가공되지 않는 카운트를 얻게 해주고 이를 이용하여 테라에서 제공되는 양자 정보 도구들을 활용하여 더 복잡한 분석을 가능하게 한다.

  5. 양자 정보 더 진보된 알고리즘과 회로 분석을 양자 컴퓨터에서 하고 싶다면, 간단한 양자 정보 작업을 할 수 있는 도구들을 사용하는 것은 중요하다. 이는 메트릭(metrics)을 추정하고, 양자 상태, 연산 및 채널을 만드는 것을 포함한다.

  6. Visualization In Terra we have many tools to visualize a quantum circuit. This allows a quick inspection of the quantum circuit to make sure it is what the user wanted to implement. There is a text, python and latex version. Once the circuit has run it is important to be able to view the output. There is a simple function (plot_histogram) to plot the results from a quantum circuit including an interactive version. There is also a function plot_state and plot_bloch_vector that allow the plotting of a quantum state. These functions are usually only used when using the statevector_simulator backend but can also be used on real data after running state tomography experiments (Ignis).

아르¶

아르는 〈공기’에 해당하고, 키스킷의 모든 구성 요소에 다양하게 사용된다. 양자 컴퓨터의 발전을 촉진하기 위해서는 더 나은 시뮬레이터, 에뮬레이터 그리고 디버거가 필요하다. 아르는 고전 프로세서들이 어느 정도까지 양자 컴퓨터를 흉내낼 수 있는지 보여 줌으로서 고전적인 방법의 한계를 알도록 해준다. 더욱이, 아르를 사용하여 현재 혹은 가까운 미래의 양자 컴퓨터의 기능들을 정확하게 확인할 수 있다. 이는 시뮬레이션의 한계를 확장하고 계산 과정에 있는 실제적인 노이즈를 시뮬레이션함으로써 가능하다.

아르는 키스킷 소프트웨어 스택을 이용하여 양자 회로를 위한 고성능 시뮬레이터 프레임워크를 제공한다. 테라에서 컴파일된 회로를 실행하는 최적화된 C++ 시뮬레이터가 있다. 아르는 또한 실제 장치에서 실행되는 동안 일어나는 오류에 대한 실제적인 잡음 시뮬레이션을 수행하는 사용자가 다양하게 수정할 수 있는 잡음 모델을 구성하는 도구를 제공한다.

키스킷 아르는 세 가지 고성능 시뮬레이터 후위 처리 장치를 포함한다:

Qasm 시뮬레이터

키스킷 회로를 이상적인 경우와 잡음이 있는 경우에서 다양하게 실행하고, 그 결과나 메모리 값을 돌려준다. 다양한 회로들을 효율적으로 시뮬레이션하는데 사용되는 여러가지 방법들은 다음과 같다:

  1. 상태벡터 - 밀집한 상태 벡터 시뮬레이션을 이용한다.

  2. 안정 장치 - 클리포스(Clifford) 회로와 잡음 모델에서만 사용할 수 있는 클리포드 안정 장치 상태 시뮬레이터를 사용한다.

  3. 확장된 안정장치 - 클리포드 게이트가 아닌 경우의 수에 따라 증가하는 안정 장치 항목으로 회로를 분해하는 근사값 시뮬레이터를 사용한다.

  4. 행렬 곱 상태 - 행렬 곱 상태 (Matrix Product State (MPS)) 시뮬레이터를 이용한다.

상태벡터 시뮬레이터

키스킷 회로를 이상적인 상태의 일회적인 연산을 하고, 응용 프로그램 후에 시뮬레이터의 상태 벡터를 돌려주는 역할을 한다.

단일 연산 시뮬레이터

키스킷 회로를 이상적인 경우의 단일 실행을 하고, 회로의 마지막 단일 행렬값을 돌려준다. 이 후위 처리 장치를 위해서 회로는 측정 메소드나 초기화 연산이 불가능 하다는 점을 유의한다.

이그니스¶

이그니스는 〈불’을 나타내고, 잡음과 오류를 관리하고, 새로운 길을 모색하는데 특화된 패키지다. 여기에는 에러를 더욱 잘 특징 짓고, 게이트를 향상 시키며, 노이즈가 있을 때 계산을 하는 동작을 포함한다. 이그니스는 양자 오류 정정을 디자인 하려는 사람들이나 단층 촬영 같은 방법을 이용한 오류 특성 파악을 공부하고자 하는 사람들, 동적 비동기화나 최적화 조절 등을 이용하여서 게이트를 더욱 잘 이용하는 법을 찾으려는 사람들에게 유용하다.

이그니스는 사용자들이 최소한의 사용자 입력 매개변수를 가지고 특정 실험을 위한 회로를 쉽게 만들 수 있는 코드를 제공한다. 이그니스 코드는 세가지 핵심적인 구성 블록을 가진다.

회로들(Circuits)

회로 모듈은 최소한의 사용자 매개 변수를 기반으로 하는 특정 이그니스 실험을 위한 회로의 목록을 만들어내는 코드를 제공한다. 이들은 이후 테라나 아르에서 실행된다.

적합 기술자들(Fitters)

이그니스 실험의 결과는 적합 기술자 모듈로 전달되는데, 여기서는 결과가 분석되고, 실험을 기술하는 물리 모델에 따라 적합하게 맞춘다. 적합 기술자는 자료를 적합하게 맞춘 결과와 함께 그려주고 매개 변수의 목록을 출력한다.

필터들(Filters)

어떤 이그니스 실험에 대해서는 적합 기술자는 Filter(필터) 객체를 출력할 수도 있다. 필터들은 이그니스 실험의 교정 결과를 이용하여 다른 실험의 오류를 줄이는데 사용될 수도 있다.

키스킷 이그니스는 세 가지 유형의 실험으로 구분될 수 있다:

특성화

특성화 실험은 게이트들에서 ZZ 상호작용률 같은 해밀톤(Hamiltonian) 매개 변수나 잡음 매개 변수(T1, T2-star, T2) 같은 변수를 측정하고 오류를 조절 하도록 설계되었다.

증명

증명 실험은 게이트나 작은 회로의 성능을 입증하기 위해 설계되었다. 여기에는 상태 및 과정 단층 촬영, 양자 볼륨 그리고 임의 벤치마킹 (randomized benchmarking, RB) 가 포함된다. 이런 실험들은 게이트의 정확성과 같은 성능 메트릭을 결정하기 위한 정보를 제공한다.

완화

완화 실험은 같은 후위 처리 장치에서 실행된 임의의 결과 집합에 적용되는 완화 루틴을 만들기 위해 분석용 교정 회로를 실행한다. 이그니스 코드는 교정 측정을 수행하는 회로 목록을 생성할 것이다. 이런 측정의 결과들은 적합 기술자로 처리되고 다른 결과에 완화 루틴을 적용하는데 사용될 필터를 출력할 것이다.

아쿠아¶

아쿠아는 〈물’을 나타내는 생명의 구성 요소이다. 양자 컴퓨팅이 기대되는 수준까지 상용화되려면, 실생활에 사용되는 응용 프로그램을 찾아야 한다. 아쿠아는 양자 컴퓨터를 위한 알고리즘이 만들어지는 곳이다. 이러한 알고리즘은 양자 컴퓨팅을 위한 응용 프로그램을 만드는데 사용된다. 화학, 최적화, 금융, AI 분야에 종사하는 전문가들이 아쿠아를 사용하게 되는데, 특정 연산 작업을 위한 가속화 장치로 양자 컴퓨터를 사용하는 사람들에게 유익할 것이다.

양자 컴퓨팅의 힘으로 해결하기 좋은 문제들은 화학, 인공 지능(AI), 최적화 및 금융 부분 등 여러 도메인에서 확인되었다. 하지만, 양자 컴퓨팅은 매우 특화된 기술들을 필요로 한다. 여러 다양한 레벨의 소프트웨어 스택에서 양자 컴퓨팅을 사용하거나 기여하고자 하는 많은 전문가 집단의 요구를 충족시키기 위해 키스킷 아쿠아(Aqua)를 제공한다.
