Qiskit Kurulumu Yapmak¶

Gereksinimler¶

Qiskit, Python 3.5 veya daha üst sürümleri desteklemektedir. Fakat, hem Python hem de Qiskit devamlı evrilen ekosistemler olduğu için herhangi birinin yeni sürümünün çıkması durumunda uyumluluk problemleri ortaya çıkabilir.

Eğer Python’u güncelledikten sonra Qiskit’i yüklemekte veya kullanmakta sıkıntı yaşıyorsanız, hangi Python versiyonlarının desteklendiğini görmek için Qiskit Paket metadatası içerisinde Programming Language kısmını kontrol edin.

Bilimsel programlama için çok platformlu Python dağıtımı olan Anaconda yı kurmanızı tavsiye ederiz. Anaconda bünyesindeki Jupyter, Qiskit ile etkileşime girmek için tavsiye edilir.

Qiskit, aşağıdaki 64 bitlik sistemler üzerinde test edilmiş ve desteklenir:

  • Ubuntu 16.04 veya üzeri

  • macOS 10.12.6 veya üzeri

  • Windows 7 veya üzeri

Kurulum¶

Qiskit’i diğer uygulamalardan net bir şekilde ayırabilmek ve deneyiminizi geliştirmek için Python sanal ortamınlarını kullanmanızı tavsiye ederiz.

Ortamları kullanmanın en basit yolu Anaconda’nın içindeki conda komutunu kullanmaktır. Bir Conda ortamı Python’ın özel bir sürümünü belirtmenize ve kütüphaneler kurmanıza imkan sağlar. Çalışmak istediğiniz klasör içinde bir terminal penceresi açın.

It is preferred that you use Anaconda prompt installed with the Anaconda. All you have to do is create a virtual environment inside Anaconda and activate the environment. These commands can be run in Anaconda prompt irrespective of Windows or linux machine.

Yalnızca Python yüklü olan bir minimal ortamı oluşturun.

conda create -n name_of_my_env python=3

Activate your new environment.

conda activate name_of_my_env

Not

If you are using conda versions prior to 4.6, use: For Linux or macOS: source activate name_of_my_env For Windows: activate name_of_my_env

Sonra, Terra, Aer, Ignis, ve Aqua’yı içeren Qiskit paketini yükleyin.

pip install qiskit

Not

Qiskit 0.13.0’dan başlamak üzere, Linux üzerinde ön derlenmiş iki terimli’den (precompiled binary) qiskit-aer yüklemek için pip 19 veya daha yeni bir versiyonu gerekmektedir. Eğer pip 19 yüklü değilse, pip install -U pip komutunu kullanarak yükleyebilirsiniz. pip 19 veya daha yeni bir versiyonu olmadığı durumda bu komut qiskit-aer’ı, aer’ı yerel ve örtülü olarak derlemeyi deneyecek olan sdist’ten (kaynak dağıtımı) yüklemeye çalışacaktır.

Eğer paketler doğru şekilde yüklendiyse, sanal ortamınızdaki (Virtual environment) aktif paketleri görmek için conda list komutunu çalıştırabilirsiniz.

If you intend to use visualization functionality or jupyter notebooks it is recommended to install qiskit with the visualization extra requirements

pip install qiskit[visualization]

It is worth point at if you’re a zsh user (which is the default shell on newer versions of macOS) you’ll need to put qiskit[visualization] in quotes:

pip install 'qiskit[visualization]'

Not

Kullanmak istediğiniz Qiskit paketlerini yükledikten ve doğruladıktan sonra, çalışmaya başlamak için Python ile onları ortamınıza aktarın.

import qiskit

Not

Eğer en son güncellenen Qiskit kodunu geliştirerek veya koda katkıda bulunarak Qiskit topluluğuna katkı sağlamak istiyorsanız, Build Qiskit packages from source a bakın.

IBM Kuantum Sistemlerine Erişim¶

IBM Quantum, Qiskit ile IBM Kuantum Deneyimi aracılığıyla birçok gerçek kuantum bilgisayarı ve yüksek performanslı klasik hesaplama simülatörü sunar. Qiskit ortamınızı IBM kuantum sistemlerine iş göndermek üzere kurmak için şu adımları takip edin.

Not

Qiskit 0.11’in piyasaya sürülmesiyle, eğer önceden IBM Quantum kimlik bilgilerinizi yerel olarak kaydettiyseniz, IBM Quantum Experience kimlik bilgilerinizi yenilemeniz gerekebilir, ki böylece yeni IBM Quantum Experience V2’yi kullanabilin. İnceleyin: Updating your IBM Quantum Experience Credentials.

Hesabınızı yapılandırmak için, bir API anahtarı içeren yerel bir konfigürasyon dosyası oluşturursunuz

1. Ücretsiz bir IBM Quantum Experience hesabı oluşturun.

2. Hesap ayarlarınızı görüntülemek için My Account’a gidin.

3. Panonuza token’ı kopyalamak için Copy token a tıklayın. Bu API token’ını geçici olarak favori metin editörünüze kopyalayın. Böylece daha sonra bir hesap konfigürasyon dosyası oluşturmak için kullanabilirsiniz.

4. qiskitrc olarak adlandırılan konfigürasyon dosyasında sonraki kullanıma uygun API token’ınızı yerel olarak depolamak için aşağıdaki komutları yürütün. Metin editörünüzde depoladığınız API token değerini MY_API_TOKEN ile değiştirin.

from qiskit import IBMQ
IBMQ.save_account('MY_API_TOKEN')

Çoklu IBM Kuantum hesap kimlik bilgilerinin nasıl yönetildiği gibi daha fazla detay için, bu eğitsel başlıklı The IBM Quantum Account na başvurun.

Hangi Versiyonun Yüklü Olduğunu Öğrenmek¶

Qiskit paketi farklı elementlerin konstelasyonunu içerdiğinden dolayı, qiskit.__version__ u çalıştırarak versiyonu kolay bir şekilde yazdırmak sadece qiskit-terra paketine uygun sürümü döndürerek yanıltabilir. Bunun sebebi Python’daki qiskit namespace’inin Qiskit paketinden değil, fakat onun yerine qiskit-terra paketinin bir parçası olmasındandır.

import qiskit
qiskit.__version__

'0.14.2'

Ortamınızdaki bütün Qiskit elementlerinin sürümlerini görmek için __qiskit_version__ özelliğini kullanabilirsiniz. Örneğin, aşağıdaki komudu çalıştırmak yüklenen her Qiskit paketine uygun versiyonu içeren bir sözlük getirir.

qiskit.__qiskit_version__

{'qiskit-terra': '0.14.2',
 'qiskit-aer': '0.5.2',
 'qiskit-ignis': '0.3.3',
 'qiskit-ibmq-provider': '0.7.2',
 'qiskit-aqua': '0.7.3',
 'qiskit': '0.19.6'}

Tüyo

Eğer bir sorun bildiriyorsanız veya bir iş için yüklü Qiskit versiyonlarınızı paylaşmanız gerekiyorsa __qiskit_version__ değişkenini kullanın.
